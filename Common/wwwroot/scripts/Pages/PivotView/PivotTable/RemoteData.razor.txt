<ActionDescription>
<p>
    This sample demonstrates the basic rendering of a pivot table bound to JSON or CSV data retrieved from a remote server.
  </p>
  <p>
    Syncfusion Blazor Pivot Table supports both JSON and CSV data sources. The <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.PivotView.PivotViewDataSourceSettings-1.html#Syncfusion_Blazor_PivotView_PivotViewDataSourceSettings_1_DataSource">DataSource</a> property in <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.PivotView.PivotViewDataSourceSettings-1.html">PivotViewDataSourceSettings</a> can be assigned with the result of <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.DataManager.html">DataManager</a> to bind remote data efficiently.
  </p>
  <p>
    The <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.DataManager.html" aria-label="Class reference of DataManager">SfDataManager</a> acts as an interface between the service endpoint and the pivot table. To properly interact with the service endpoint and retrieve the required data, the following minimal configuration is necessary:
  </p>
  <ul>
    <li>
      <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.DataManager.html#Syncfusion_Blazor_DataManager_Url" aria-label="Class reference of Url property in DataManager">SfDataManager->Url</a> - Defines the service endpoint URL to fetch data.
    </li>
    <li>
      <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.DataManager.html#Syncfusion_Blazor_DataManager_Adaptor" aria-label="Class reference of Adaptor property in DataManager">SfDataManager->Adaptor</a> - Defines the adaptor option. <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.Adaptors.html#Syncfusion_Blazor_Adaptors_WebApiAdaptor" aria-label="Class reference of WebApiAdaptor Property in Blazor Adaptor">WebApiAdaptor</a> is commonly used for remote binding.
    </li>
  </ul>
  <p>
    The adaptor is responsible for processing requests and responses from/to the service endpoint. Syncfusion provides several predefined adaptors:
  </p>
  <ul>
    <li>
      <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.Adaptors.html#Syncfusion_Blazor_Adaptors_UrlAdaptor" aria-label="Class reference of UrlAdaptor Property in Blazor Adaptor">UrlAdaptor</a> - Use this to interact with any remote service. This is the base adaptor for all remote-based adaptors.
    </li>
    <li>
      <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.Adaptors.html#Syncfusion_Blazor_Adaptors_ODataAdaptor" aria-label="Class reference of ODataAdaptor Property in Blazor Adaptor">ODataAdaptor</a> - Use this to interact with OData endpoints.
    </li>
    <li>
      <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.Adaptors.html#Syncfusion_Blazor_Adaptors_ODataV4Adaptor" aria-label="Class reference of ODataV4Adaptor Property in Blazor Adaptor">ODataV4Adaptor</a> - Use this to interact with OData V4 endpoints.
    </li>
    <li>
      <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.Adaptors.html#Syncfusion_Blazor_Adaptors_WebApiAdaptor" aria-label="Class reference of WebApiAdaptor Property in Blazor Adaptor">WebApiAdaptor</a> - Use this to interact with Web APIs created under OData standards.
    </li>
  </ul>
  <p>
    In this demo, remote data binding is achieved by assigning the service data as an instance of <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.DataManager.html" aria-label="Class reference of DataManager">DataManager</a> to the <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.PivotView.PivotViewDataSourceSettings-1.html#Syncfusion_Blazor_PivotView_PivotViewDataSourceSettings_1_DataSource" aria-label="Class reference of DataSource property in PivotView">DataSource</a> property in <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.PivotView.PivotViewDataSourceSettings-1.html" aria-label="Class reference of PivotViewDataSourceSettings">PivotViewDataSourceSettings</a>. For CSV data, the service URL is directly set to the <a target="_blank" rel="noopener" class="code" href="https://help.syncfusion.com/cr/blazor/Syncfusion.Blazor.PivotView.PivotViewDataSourceSettings-1.html#Syncfusion_Blazor_PivotView_PivotViewDataSourceSettings_1_Url" aria-label="Class reference of Url property in PivotView">Url</a> property for remote data consumption.
  </p>
  <p>
    For detailed information about remote data binding configuration and best practices, refer to the <a target="_blank" rel="noopener" href="https://blazor.syncfusion.com/documentation/pivot-table/data-binding/#remote-data" aria-label="Blazor PivotTable Remote Databinding documentation">remote data binding documentation</a>.
  </p>
</ActionDescription>

@code{
    public string CurrentDataType { get; set; } = "JSON";

    private List<DropDownData> ContentTypes = new List<DropDownData>()
    {
        new DropDownData { Name = "JSON", Value = "JSON" },
        new DropDownData { Name = "CSV", Value = "CSV" }
    };

    public class DropDownData
    {
        public string Name { get; set; } = string.Empty;
        public string Value { get; set; } = string.Empty;
    }

    public class PivotProductDetails
    {
        public int Quantity { get; set; }
        public double UnitPrice { get; set; }
        public string ProductName { get; set; } = string.Empty;
        public string ShipCountry { get; set; } = string.Empty;
        public string ShipCity { get; set; } = string.Empty;
    }
}
